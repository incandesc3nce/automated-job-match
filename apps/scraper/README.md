# scraper

Это сервис, который собирает данные о вакансиях с различных источников, сохраняет их в базе данных и создает задачи на обработку для `llm-worker`. Вакансии собираются в полночь каждый день с помощью планировщика задач `cron` и моментально при запуске в Docker. В настоящее время реализованы адаптеры для `hh.ru` и `getmatch.ru`, но вы можете добавить свои адаптеры, создав новый класс, который наследуется от JobSourceAdapter, и реализовать методы `fetchJobs` и `normalize`.

## Технологии

- TypeScript
- Bun
- PostgreSQL
- Drizzle ORM
- BullMQ
- cheerio для парсинга HTML
- cron для планирования задач

## Локальная разработка

1. Убедитесь, что у вас запущены все необходимые сервисы (Redis и PostgreSQL) с помощью `docker-compose up` или они запущены локально.
2. Перейдите в директорию `apps/scraper` и установите зависимости:

```bash
cd apps/scraper
bun install
```

3. Создайте файл `.env` в корне директории `apps/scraper` и добавьте переменные окружения из `docker/env-example/scraper.env`.
4. Запустите приложение приложение:

```bash
bun start
```

Приложение будет запущено и начнет собирать вакансии с указанных источников. Вы можете проверить его работу, посмотрев в базу данных или посмотрел логи приложения. Вакансии будут собираться в полночь каждый день, а также при каждом запуске приложения в Docker.
