# llm-worker

Это сервис, который отвечает за обработку запросов к LLM, векторизацию текста резюме и вакансий, сокращение описания вакансии и подборки рекомендаций по вакансиям. Он отправляет `API` оповещения о новых подборках через механизм Redis pub/sub. `llm-worker` обрабатывает задачи на запрос к LLM из очереди, которые ему отправляют `scraper` и `API`.

## Технологии

- TypeScript
- Bun
- PostgreSQL
- Drizzle ORM
- Redis
- BullMQ
- LMStudio для локальной LLM

LMStudio можно заменить на любой другой "provider" LLM, который имеет API для генерации текста и эмбеддингов, и добавить его в папку `/providers` с реализацией интерфейса `LLMProvider`.

## Локальная разработка

1. Убедитесь, что у вас запущены все необходимые сервисы (Redis и PostgreSQL) с помощью `docker-compose up` или они запущены локально.
2. Перейдите в директорию `apps/llm-worker` и установите зависимости:

```bash
cd apps/llm-worker
bun install
```

3. Создайте файл `.env` в корне директории `apps/llm-worker` и добавьте переменные окружения из `docker/env-example/llm-worker.env`.
4. Запустите приложение в режиме разработки:

```bash
bun run dev
```

Приложение будет запущено и будет обрабатывать задачи из очереди. Вы можете проверить его работу, отправив задачи на генерацию эмбеддингов или сокращение описания вакансии через `API` или `scraper`.
